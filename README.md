# Kalashnikov ML Boosting Pipeline

Этот проект представляет собой автоматизированный ML-пайплайн для прогнозирования производства продукции на основе данных о потреблении материалов и цехах. Проект переведен из Jupyter Notebook в модульную структуру на Python с поддержкой кеширования этапов.

## 🚀 Основные возможности
- **Модульность**: Каждый этап (обработка, очистка, фичи, обучение) — это отдельная функция.
- **Кеширование**: Этапы пропускаются, если входные данные и код функции не изменились (экономит время на предобработке).
- **Настройка**: Все параметры (колонки, пороги выбросов, гиперпараметры Optuna) вынесены в `config.yaml`.
- **Три модели**: Автоматический подбор гиперпараметров через Optuna для XGBoost, LightGBM и CatBoost.
- **Управление зависимостями**: Использование `uv` для быстрого и надежного окружения.

---

## 📂 Структура проекта
- `main.py` — точка входа для запуска пайплайна.
- `config.yaml` — конфигурационный файл со всеми настройками.
- `src/` — исходный код этапов:
  - `stages.py` — логика обработки данных и визуализации.
  - `cache_utils.py` — система проверки хешей для пропуска этапов.
  - `models_utils.py` — обучение моделей и интеграция с Optuna.
- `data/` — данные (raw, interim, processed).
- `models/` — сохраненные обученные модели (.pkl, .cbm).
- `reports/` — графики корреляции, важности признаков и CSV с метриками.
- `pipeline_state.json` — файл состояния для системы кеширования.

## 🛠 Установка и запуск

Для работы требуется установленный [uv](https://github.com/astral-sh/uv).

### 1. Установка зависимостей
```bash
uv sync
```

### 2. Запуск всего пайплайна
Запустить все этапы от объединения данных до вывода метрик:
```bash
uv run python main.py --stage 1
```

### 3. Запуск с конкретного этапа
Если вы хотите пропустить предобработку и начать сразу с обучения (этап 6):
```bash
uv run python main.py --stage 6
```

### 4. Обучение конкретной модели
```bash
uv run python main.py --stage 6 --model xgboost
```

## 📝 Описание этапов (Stages)
1. **Combine**: Объединение таблиц потребления и производства по Цеху и Дате.
2. **Clean**: Удаление пустых значений и ненужных технических колонок.
3. **Outliers**: Фильтрация выбросов по целевой переменной (Z-score).
4. **Features**: Создание признаков сезонности (sin/cos месяца и дня, год, выходные).
5. **Visualize**: Генерация матрицы корреляции в папке `reports/`.
6. **Training**: Подбор гиперпараметров Optuna и сохранение 3-х моделей в `models/`.
7. **Evaluation**: Загрузка моделей и расчет RMSE, MAE, R2 на тестовой выборке.
8. **Plots**: Построение графиков важности признаков (Feature Importance).

## ⚙️ Настройка (config.yaml)
Вы можете менять поведение пайплайна без правки кода:
- `outlier_threshold`: Чувствительность к выбросам (по умолчанию 3.0).
- `optuna_trials`: Количество попыток подбора параметров (установите 50-100 для качества).
- `drop_columns`: Список колонок, которые будут удалены перед обучением.
- `test_size` / `val_size`: Пропорции разделения выборки.
